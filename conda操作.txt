conda代理
    设置代理 先设置free在设置main，这样main在上面
       	conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
       	conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
    设置启动设置好的国内镜像源
       	conda config --set show_channel_urls yes

    查看是否添加上了源
        conda config --show
        channel默认是defaults,修改之后就不一样了

    删除镜像
    	conda config --remove channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
        conda config --remove channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/


环境管理
	linux初始化
		conda init
	进入环境
		conda activate 环境名

	退出环境
		conda deactivate

	创建虚拟环境:  conda create -n 环境名 python=3.6
		创建完成后目录文件夹会生成在Anaconda3的安装目录envs下边
		linux会创建在 home/xxx/.conda/envs/ 目录下

	删除某个环境
		conda remove -n 环境名 --all

	列出所有可用环境
		conda env list
		conda info --envs

	从旧的环境克隆出一个新环境
		conda create -n 环境名 --clone oldname

	导出环境配置(进入虚拟环境中使用)
		conda env export > environment.yml

	用导出的配置生成一个新环境
		conda env create -f environment.yml

    删除缓存
        conda clean -a


包管理
	conda list 					列举当前环境下的所有包
	conda list -n packagename 	列举某个特定名称包
	conda install packagename 	为当前环境安装某包
	conda install packagename=11.3 指定版本 pip是两个等号
	conda install -n envname packagename 为某环境安装某包
	conda search packagename 	搜索某包
	conda updata packagename 	更新当前环境某包
	conda update -n envname packagename 更新某特定环境某包

	conda remove packagename 删除当前环境某包
	pip安装的要这样卸载
		pip uninstall packagename

	conda remove -n envname packagename 删除某环境某包

	conda本身和anaconda、python本身也算包
	conda update -n base conda	主环境conda
	conda update conda

	conda update anaconda
	conda update python
	conda install python=3.8

# 给powershell添加conda路径问题
#	问题1(已解决)：
#		pycharm默认的终端是powershell，进入的就是你的环境，即使没有提示，不过添加conda路径之后默认进入base，要手动进入你的环境中
#		解决办法：设置conda不自动激活base环境	conda config --set auto_activate_base false
#
#		右键运行代码执行的是 `D:\Anaconda3\envs\ai\python.exe E:/ai/test.py`，没有任何问题
#
#	问题2(已解决):
#		vscode右键运行默认是powershell，在没有添加conda路径之前，不能进入conda环境，右键命令是 `& conda run -n ai --no-capture-output --live-stream python e:/ai/test.py`
#		可以正常运行，而在进入conda环境之后，这个命令会报错，因为 `run -n ai` 不能在 ai 环境中运行，要在base中运行，所以要回到base才能用(cmd也要这样)，所以不建议添加conda到powershell
#
#		解决办法： 在vscode设置中搜索python.terminal.activateEnvironment，取消勾选就不会自动进入虚拟环境了(PS:上面的命令在linux上即使进入虚拟环境也不报错)
